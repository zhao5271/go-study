# 02 基础类型（转换、格式化、表达式）

> 更新于：2026-03-20  
> 目标：把“显式转换、`fmt/strconv`、表达式运算”整理成后端日常可直接复用的基础能力。

## TL;DR（可放入 progress/context-pack）
- Go 不做隐式数值类型转换；`int`、`int64`、`float64` 混用必须显式转换。
- 显式转换不等于安全转换：`float64 -> int` 会截断，`int64 -> int` 可能溢出。
- 数字和字符串互转优先用 `strconv`；`string(65)` 得到的是 `"A"`，不是 `"65"`。
- `fmt.Printf` 直接输出，`fmt.Sprintf` 返回字符串，`fmt.Fprintf` 写入 `io.Writer`。
- 表达式最常见的坑：整数除法截断、短路逻辑理解错误、位运算省括号、边界校验漏掉。
- 这些点在分页、日志、错误码、权限位、参数解析里天天都会遇到。

## 知识点 1：显式类型转换

### 一句话定义
Go 里不同类型之间不会自动转换；你必须用 `T(x)` 明确表达“我要把它当成另一个类型来用”。

### 为什么重要
后台管理 API 里，query 参数是字符串，数据库里常见 `int64`，业务里常见 `int` 做分页、长度、下标；这几个边界天天都要跨。

### 重难点拆解
- `int` 与 `int64` 不能直接相加/比较，必须显式转换。
- `float64 -> int` 会向 0 截断，不会四舍五入。
- `int64 -> int` 在 32 位平台上可能溢出，不能默认它总是安全。
- 无类型常量（如 `const x = 1`）比变量更“宽松”；一旦落成变量，就要遵守具体类型规则。

### 业务场景落地
- `COUNT(*)` 扫描后经常是 `int64`，而分页逻辑常用 `int`
- 金额、比例、评分一旦涉及浮点转整数，如果你没明确“截断还是四舍五入”，结果很容易错
- 前端传来的 `"20"` 必须先解析，再参与业务计算

### 代码示例
```go
package main

import (
	"fmt"
	"math"
)

func main() {
	x := 42
	var big int64 = 1
	sum := int64(x) + big
	fmt.Printf("sum=%d type=%T\n", sum, sum) // Output: sum=43 type=int64

	f := 1.9
	fmt.Printf("int(1.9)=%d\n", int(f))                // Output: int(1.9)=1
	fmt.Printf("round(1.9)=%d\n", int(math.Round(f))) // Output: round(1.9)=2
}
```

### 练习题 1
写一个函数，把 `int64 total` 安全转换成 `int` 返回；如果超出当前平台 `int` 的范围，就返回错误。

**验收标准**
- 返回 `(int, error)`
- 至少考虑 32 位平台溢出

**参考答案**
```go
package main

import (
	"fmt"
	"math"
	"strconv"
)

func int64ToInt(v int64) (int, error) {
	if strconv.IntSize == 32 && (v < math.MinInt32 || v > math.MaxInt32) {
		return 0, fmt.Errorf("int overflow: %d", v) // Output: 错误信息内容固定；数字随输入变化
	}
	return int(v), nil
}
```

## 知识点 2：`fmt` / `strconv`

### 一句话定义
`fmt` 负责格式化输出，`strconv` 负责字符串与基础类型之间的解析和转换。

### 为什么重要
你写日志、审计信息、错误提示、query 解析、环境变量读取时，几乎一定会用到这两个包。

### 重难点拆解
- `fmt.Printf` 写到标准输出；`fmt.Sprintf` 返回字符串，适合拼日志和错误信息。
- `%q` 很适合调试字符串；它会把换行、引号、反斜杠等转义显示出来。
- `strconv.Atoi` / `Itoa` 处理十进制 `int` 最方便；更大范围要用 `ParseInt` / `FormatInt`。
- `string(65)` 是把 65 当 rune 转成 `"A"`，不是把数字转成十进制字符串。

### 业务场景落地
- 解析分页参数：`page, err := strconv.Atoi(r.URL.Query().Get("page"))`
- 构造审计日志：`fmt.Sprintf("user=%s action=%s", user, action)`
- 调试请求体和错误值时，用 `%q` / `%T` 很省时间

### 代码示例
```go
package main

import (
	"fmt"
	"strconv"
)

func main() {
	n := 65
	fmt.Printf("string(n)=%q\n", string(n))      // Output: string(n)="A"
	fmt.Printf("Itoa(n)=%q\n", strconv.Itoa(n))  // Output: Itoa(n)="65"

	v, err := strconv.Atoi("65")
	if err != nil {
		fmt.Printf("Atoi error=%v\n", err) // Output: 输出可能变化/不固定：错误文本随输入和实现变化
		return
	}
	fmt.Printf("Atoi(\"65\")=%d\n", v) // Output: Atoi("65")=65
}
```

### 练习题 2
把字符串 `"18080"` 转成端口号 `int`；如果解析失败，返回 `invalid port` 包装后的错误。

**验收标准**
- 使用 `strconv.Atoi`
- 错误要带上下文

**参考答案**
```go
package main

import (
	"fmt"
	"strconv"
)

func parsePort(s string) (int, error) {
	port, err := strconv.Atoi(s)
	if err != nil {
		return 0, fmt.Errorf("invalid port: %w", err) // Output: 错误信息随输入变化；保留底层错误语义
	}
	return port, nil
}
```

## 知识点 3：表达式与运算符

### 一句话定义
表达式是“会算出值的代码”；在 Go 里，最常见问题不是语法不会写，而是你对结果边界想错了。

### 为什么重要
分页 offset、权限判断、空指针保护、条件短路，全都依赖表达式语义；这里错了，业务就会悄悄算错。

### 重难点拆解
- `5/2 == 2`，因为整数除法直接截断小数。
- `&&` 和 `||` 是短路运算；前半部分已能决定结果时，后半部分不会执行。
- 位运算建议总是加括号：`(mask & PermRead) != 0`。
- 参数边界校验要尽量早做，别把非法值带进后面的计算。

### 业务场景落地
- 分页：`offset := (page - 1) * size`
- 权限：`(mask & PermExport) != 0`
- nil 安全：`if p != nil && *p > 0 { ... }`

### 代码示例
```go
package main

import "fmt"

type Perm uint64

const (
	PermRead Perm = 1 << iota
	PermWrite
	PermExport
)

func main() {
	fmt.Printf("5/2=%d\n", 5/2)                 // Output: 5/2=2
	fmt.Printf("5/2=%.1f\n", float64(5)/2)     // Output: 5/2=2.5

	var p *int
	if p != nil && *p > 0 {
		fmt.Println("p>0") // Output: 不会打印，因为 p==nil 且 && 短路
	} else {
		fmt.Println("short-circuit ok") // Output: short-circuit ok
	}

	mask := PermRead | PermExport
	fmt.Printf("hasRead=%t\n", (mask&PermRead) != 0)   // Output: hasRead=true
	fmt.Printf("hasWrite=%t\n", (mask&PermWrite) != 0) // Output: hasWrite=false
}
```

### 练习题 3
实现 `calcOffset(page, size int) (int, error)`：
- `page < 1` 报错
- `size < 1 || size > 100` 报错
- 正常时返回 `(page - 1) * size`

**验收标准**
- 先做边界校验，再计算
- 返回 `(int, error)`

**参考答案**
```go
package main

import "fmt"

func calcOffset(page, size int) (int, error) {
	if page < 1 {
		return 0, fmt.Errorf("page must be >= 1") // Output: page must be >= 1（当 page 非法时）
	}
	if size < 1 || size > 100 {
		return 0, fmt.Errorf("size must be 1..100") // Output: size must be 1..100（当 size 非法时）
	}
	return (page - 1) * size, nil
}
```
